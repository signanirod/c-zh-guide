# c-zh-guide

## 项目简介

c-zh-guide 是一个用于归档和发布多个独立 HTML 页面的仓库。这些页面不针对任何单个域名或具体网站，而是以中文内容为主，涵盖各类指南、参考信息或实用工具页面。

本仓库旨在提供一个集中存放 HTML 静态页面的空间，便于用户直接通过 GitHub Pages 或其他静态托管服务访问。

## 目录说明

| 目录/文件 | 说明 |
|-----------|------|
| `pages/`  | 存放所有独立 HTML 页面的目录 |
| `assets/` | 页面所需的公共资源，如样式表、脚本、图片等 |
| `index.html` | 仓库首页（可选） |

每个 HTML 页面均为独立自包含文件，部分页面可能引用 `assets/` 中的公共资源。

## 页面归档说明

- 每个页面均以 `.html` 后缀命名，文件名应简洁明了，反映页面内容主题。
- 页面内容以中文书写为主，结构清晰，便于阅读和归档。
- 所有页面均为静态 HTML，不依赖后端服务。
- 页面之间无强关联，可独立访问。

## 维护说明

- 欢迎提交 Pull Request 添加新页面或改进现有页面。
- 新增页面请放置在 `pages/` 目录下，并确保文件名不与已有页面冲突。
- 请保持页面风格简洁，避免引入过多外部依赖。
- 如有公共资源更新，请放在 `assets/` 目录下，并在页面中正确引用。
- 本仓库将不定期整理和更新页面内容。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 目录下添加或修改 HTML 页面。
3. 提交更改并推送至 GitHub。
4. 若启用 GitHub Pages，可通过 `https://<用户名>.github.io/c-zh-guide/pages/<页面文件名>` 访问对应页面。
